. 
Pastikan Kamu Menggunakan API Key Valid (Authentication Issue)
• Dari tool call saya, endpoint seperti Topic_Time_Series dan Topic_Posts mengembalikan error "You are not authenticated" karena placeholder "Your_API_Key_Here". LunarCrush memerlukan API key premium (Pro tier atau lebih tinggi) untuk akses full data real-time.
• Cara Fix:
• Daftar di LunarCrush API Dashboard dan dapatkan API key gratis (untuk basic) atau upgrade ke Pro ($99/bulan untuk lebih banyak calls dan data historis).
• Masukkan key di header request: Authorization: Bearer YOUR_ACTUAL_API_KEY.
• Test key di API Playground untuk endpoint /v2/topic/{topic}/posts/1d atau /v2/topic/{topic}/time-series/posts_active/1w.
• Jika masih error, cek rate limit (100 calls/minute untuk Pro; reset setiap 24 jam). Gunakan key baru jika expired.

2. 
Gunakan Contract Address (CA) untuk Query Spesifik, Bukan Ticker Saja
• Ticker "$beat" atau "$BEAT" ambigu, jadi LunarCrush default ke token lama (MetaBeat). Gunakan CA lengkap untuk Audiera $BEAT: 0xcf3232B85b43BCa90E51D38cc06Cc8bB8C8A3E36.
• Cara Fix:
• Query endpoint /v2/topic/0xcf3232B85b43BCa90E51D38cc06Cc8bB8C8A3E36 (ganti {topic} dengan CA).
• Contoh curl request (gunakan Postman atau script):
curl -H "Authorization: Bearer YOUR_API_KEY" \
"https://api.lunarcrush.com/v2/topic/0xcf3232B85b43BCa90E51D38cc06Cc8bB8C8A3E36?data_points=24&interval=1h"
• Ini akan return data mentions/posts active per jam (misal: posts_active: 806, interactions: 297K dari data terbaru).
• Jika CA belum terindeks, coba query dengan nama proyek: /v2/topic/audiera atau /v2/topic/audiera beat.
• Interval & Metrics: Gunakan interval=1d untuk daily, 1w untuk weekly. Metrics: posts_active (mentions), interactions (engagements), contributors_active (unique creators). Contoh dari data saya:
• Posts Active (mentions): 806 (24h terakhir, naik 291% dari average).
• Interactions: 297,599 (termasuk likes/retweets).
• Contributors: 432 (unique users, naik 51% hari ini).
• Output Contoh (dari tool call saya untuk CA ini):
• Time-series 1 minggu: Posts_active naik dari ~60 (awal minggu) ke 806 (hari ini), dengan spike di jam 06:00-17:00 UTC (saat listings diumumkan).
• Total mentions: ~6,072 di X, 2 di YouTube (fokus di Twitter karena pump futures).

3. 
Cek dan Refresh Data dengan Endpoint Khusus
• LunarCrush punya delay untuk token baru (seperti Audiera $BEAT yang baru hot di 13-14 Nov 2025). Data saya tunjukkan mentions mulai naik dari 60 (6 Nov) ke 806 (14 Nov), tapi jika di API kamu masih 0, itu karena cache atau indexing lambat.
• Cara Fix:
• Gunakan endpoint Search untuk keywords: /v2/search?q=audiera beat&limit=100 (return posts terkait, termasuk mentions).
• Contoh hasil: Banyak post tentang "Audiera $BEAT" dengan 100+ engagements dari akun seperti 

 @Audiera_web3 (41K engagements), 

 @Phemex_official (15K), 

 @BinanceFutures (21K).
• Gunakan Topic_Posts untuk posts teratas: /v2/topic/audiera/posts/1d?limit=50&from_date=2025-11-13&to_date=2025-11-14.
• Ini return 50+ posts real-time (misal: Post dari 

 @Audiera_web3 tentang listings, dengan 19K engagements).
• Time-Series untuk Historis: /v2/topic/0xcf3232B85b43BCa90E51D38cc06Cc8bB8C8A3E36/time-series?data_points=168&interval=1h&metrics=posts_active.
• Data dari tool saya: Spike tinggi di 13 Nov (mentions ~255-297 per jam selama pump, total 806/24h).
• Jika masih 0, tunggu 24-48 jam (LunarCrush update setiap jam, tapi token baru butuh waktu). Atau force refresh dengan multiple queries.

4. 
Troubleshoot Umum dan Tips Tambahan
• Jika Data Masih 0:
• Token Belum Terdaftar: Audiera $BEAT mungkin belum diakui sebagai "cryptocurrency" di LunarCrush (cek di /v2/list/cryptocurrencies – dari tool call, tidak muncul di top 10, jadi indexing lambat). Sarankan submit token ke LunarCrush support via [support@lunarcrush.com] dengan CA dan nama proyek.
• Rate Limit atau Error: Cek response header untuk "Rate Limit Remaining". Jika 0, tunggu reset (1 menit atau 24 jam). Gunakan key berbeda jika multiple users.
• Fallback Tools: Jika LunarCrush lambat, cross-check dengan:
• Santiment: Query "beat" atau CA di santiment.net – mereka punya social volume lebih cepat untuk BNB Chain.
• CoinGecko API: /coins/audiera untuk mentions/social data (gratis, tapi kurang detail).
• Twitter API: Gunakan Twitter's API v2 untuk search "audiera beat" with tweet.fields=public_metrics untuk count mentions (butuh Twitter dev account).
• Dune Analytics: Query dashboard BNB Chain untuk volume mentions (gratis, on-chain focused).
• Versi API: Pastikan gunakan v2 (terbaru). Dokumentasi: lunarcrush.com/developers/api/docs.
• Optimasi Query:
• Tambah param data_points=24 untuk 24h data.
• Gunakan sort=mentions di Search untuk top posts.
• Untuk real-time, poll setiap 5-10 menit (hindari spam untuk hindari ban).
• Kenapa Ini Terjadi?: LunarCrush prioritaskan token besar (BTC, ETH). Token seperti $BEAT (micro-cap awalnya) butuh volume tinggi dulu untuk terindeks akurat. Dari data saya, sekarang sudah ada 806 mentions – coba query ulang sekarang!
